## Project Overview

**NewsNow** is a real-time news aggregation platform that scrapes and displays trending news from 30+ sources (Baidu Trending, Bilibili, GitHub Trending, HackerNews, etc.). It features GitHub OAuth login, intelligent caching, and MCP server support.

- **Repository**: https://github.com/ourongxing/newsnow
- **Deployment**: Cloudflare Pages, Vercel, Node.js, or Bun
- **Target**: Chinese news sources (demo version; English support planned)

## Tech Stack

| Layer | Technology |
|-------|-----------|
| Frontend | React 19, TanStack Router, Jotai (state), UnoCSS |
| Backend | Nitro (Node/Cloudflare/Vercel/Bun), H3 |
| Database | Better-sqlite3 (local), Cloudflare D1 (edge) |
| Scraping | Cheerio, fast-xml-parser |

| Testing | Vitest |

## Architecture Patterns

### Frontend (React + Vite)

- **Routing**: TanStack Router with file-based routing in `src/routes/`
- **State**: Jotai atoms (not Redux; atomic state machine pattern)
- **Styling**: UnoCSS utilities (Tailwind-like, compile-time)
- **Components**: Functional + hooks; no class components
- **Auto-imports**: `unimport` plugin automatically imports from `src/hooks/`, `shared/`, `src/atoms/`
  - Common imports: `useAtom`, `useAtomValue`, `atom` (Jotai)
  - Utilities: `$` (clsx), `atomWithStorage` (persisted atoms)

### Backend (Nitro)

- **Preset Selection** (via environment):
  - `NODE_ENV`: defaults to Node.js server preset
  - `VERCEL=1`: Vercel Edge preset
  - `CF_PAGES=1`: Cloudflare Pages preset
  - `BUN=1`: Bun preset
- **API Routes**: File-based in `server/api/` (e.g., `api/s/index.ts` → `GET /api/s`)
- **Database**: db0 abstraction (configurable connector in `nitro.config.ts`)
  - Local dev: better-sqlite3 (in-process)
  - Production: Cloudflare D1, Vercel Postgres, etc.
- **Auto-imports**: Nitro auto-imports from `server/utils/` and `shared/`

### Data Sources Pattern

NewsNow is built around a **pluggable source system**:

1. **Configuration** (`shared/pre-sources.ts`):
   ```typescript
   export const originSources = {
     "bilibili": {
       name: "哔哩哔哩",
       color: "blue",
       home: "https://www.bilibili.com",
       sub: {
         "hot-search": { title: "热搜", column: "china", type: "hottest" },
       }
     }
   }
   ```

2. **Implementation** (`server/sources/bilibili.ts`):
   ```typescript
   const hotSearch = defineSource(async () => {
     const res = await myFetch("https://api.bilibili.com/...")
     return res.data.map(item => ({
       id: item.id,
       title: item.title,
       url: item.url,
       extra: { info: "...", icon: "..." }
     }))
   })
   
   export default defineSource({
     "bilibili": hotSearch,
     "bilibili-hot-search": hotSearch,
   })
   ```

3. **Registration**: After changes, run `pnpm presource` to generate `shared/sources.json`

**Key types** (`shared/types.ts`):
- `SourceID`: Discriminated union of valid source IDs (enforced by TypeScript)
- `NewsItem`: `{ id, title, url, mobileUrl?, pubDate?, extra? }`
- `SourceResponse`: `{ status: "success"|"cache", id, updatedTime, items }`

### Caching Strategy

- **TTL-based**: Default 30 minutes (configurable)
- **Interval-based**: Each source has a `refresh interval` (e.g., "update at most every 2 minutes")
- **User refresh**: Logged-in users can `latest=true` to bypass cache
- **Cache table**: `better-sqlite3` stores `{ id, items, updated }` locally

### Environment & Configuration

**Frontend** (`.env.server` loaded into Vite):
- Git OAuth: `G_CLIENT_ID`, `G_CLIENT_SECRET`, `JWT_SECRET`
- Database: `INIT_TABLE`, `ENABLE_CACHE`
- Extras: `PRODUCTHUNT_API_TOKEN`

**Backend presets** (via environment variables):
- `VERCEL=1`: Use Vercel Edge preset (serverless)
- `CF_PAGES=1`: Use Cloudflare Pages preset (Workers)
- `BUN=1`: Use Bun preset
- Default: Node.js server

Docker Compose overrides these in `docker-compose.yml`.

## Development Workflow

### Adding a New Data Source

1. **Add config** to `shared/pre-sources.ts`:
   ```typescript
   "mynews": {
     name: "My News",
     color: "purple",
     home: "https://example.com",
     column: "tech",
     type: "hottest"
   }
   ```

2. **Implement fetcher** in `server/sources/mynews.ts`:
   ```typescript
   export default defineSource(async () => {
     const html = await myFetch("https://example.com/trending")
     const items = parseHTML(html)
     return items.map(item => ({ id, title, url, extra: {...} }))
   })
   ```

3. **Regenerate sources**:
   ```bash
   pnpm presource
   ```

4. **Test locally**:
   ```bash
   pnpm dev
   # Visit http://localhost:5173, search for your source
   ```

### Modifying Components

- React functional components in `src/components/`
- Styling: Use UnoCSS classes directly (no CSS files needed for utilities)
- State: Use Jotai atoms from `src/atoms/` (hook: `useAtom(myAtom)`)
- Types: Import from `shared/types.ts` or define locally

### Database Queries

- Create DB utilities in `server/database/` or `server/utils/`
- Use `db` (auto-imported from Nitro) for queries
- Schema is initialized by `INIT_TABLE=true` on first run
- Run migrations/updates in `server/database/migrations/` (if needed)

### Testing

```bash
pnpm test                    # Run all tests
pnpm test -- --watch        # Watch mode
pnpm test -- filename.test   # Single file
```

Test files go in `test/` directory (or co-locate with `*.test.ts`).

## Code Patterns & Conventions

### Type Safety

- **SourceID**: Use the discriminated union type; never hardcode source IDs
  ```typescript
  // Good
  const sources: Record<SourceID, SourceResponse> = {...}
  
  // Bad (loses type safety)
  const sources: Record<string, SourceResponse> = {...}
  ```

- **Exhaustive checks**: TS will warn if you forget to handle a source ID

### Auto-Imports

The Vite plugin `unimport` auto-imports from:
- `src/hooks/**/*` → `useMyHook()`
- `src/atoms/**/*` → `myAtom`
- `shared/**/*` → types, utils
- `src/utils/**/*` → helper functions
- Jotai: `atom`, `useAtom`, `useAtomValue`, `useSetAtom`

**Do not manually import** from these directories; the plugin handles it.

### Styling

- **UnoCSS only**: No CSS modules, CSS-in-JS, or custom CSS for component styling
- **Utility classes**: `<div className="flex gap-4 text-lg font-bold">`
- **Global styles**: `src/styles/` for reset, animations, custom directives

### Fetch & HTTP

- Backend uses `myFetch()` (auto-imported, wraps `ofetch`)
- Frontend uses `fetch` or `useQuery` (TanStack Query)
- No axios; prefer `ofetch`/`fetch`

## Gotchas & Notes

1. **`presource` is required after source changes**: Adding a new source or modifying `pre-sources.ts` requires `pnpm presource` to regenerate `sources.json`.
2. **SourceID type safety**: Never use string IDs directly; always use the `SourceID` type.
3. **Database initialization**: Set `INIT_TABLE=true` on first deployment; can be disabled afterward.
4. **Cache is smart**: Items are served from cache if they haven't been refreshed according to their interval (not just TTL).
5. **Secrets in `.env.server`**: Never commit `.env.server` with real credentials; use `.env.example` or doc env vars needed.
6. **Cloudflare-specific**: If using CF_PAGES preset, some Node APIs may not be available; test edge cases.
7. **Favicon generation**: Icons are auto-scraped and cached during `presource`; internet required.

## Common Tasks

### Debug a source that's not fetching
1. Check if source is in `shared/pre-sources.ts`
2. Check if `server/sources/xyz.ts` exports the right source ID
3. Run `pnpm presource` to regenerate
4. Check browser console and `pnpm dev` terminal for errors
5. Verify the source URL in `server/sources/xyz.ts` is correct and accessible

### Add GitHub OAuth
1. Create GitHub OAuth app at https://github.com/settings/applications/new
2. Set redirect URL to `https://yourdomain.com/api/oauth/github`
3. Add `G_CLIENT_ID`, `G_CLIENT_SECRET`, `JWT_SECRET` to env
4. Logic is in `server/api/oauth/github.ts`

### Cache isn't working
1. Check `ENABLE_CACHE=true` in env
2. Verify database is initialized (`INIT_TABLE=true` on first run)
3. Check `server/database/cache.ts` for cache queries
4. Clear cache: Delete `.data/` directory (local) or D1 table (Cloudflare)

### Add a new UI feature
1. Create component in `src/components/`
2. Add hook if needed in `src/hooks/`
3. Use Jotai for state (`src/atoms/`)
4. Style with UnoCSS utilities
5. Add to route in `src/routes/`
6. Test with `pnpm dev`
